## Migration Strategy

+ 6個R
	+ 重新託管(Lifting shift)
		+ ==僅移動應用程式==，不涉及修改
	+ 重組平台(精進平移)
		+ 涉及一些雲端最佳化作業以達成實際效益
	+ 淘汰
		+ 移除不需要的應用程式
	+ 保留
		+ 保留在來源環境中重要的業務程式
			+ 可能是需要延遲一陣子的程式
			+ 需要重大重構的程式
	+ 重新購買
		+ 從傳統授權傳轉換到Saas
		+ ==轉換至不同的產品==
	+ 重構
		+ 使用cloud native feature來重新設計應用程式架構
## AWS Snow

+ AWS SnowCone
	+ 小型邊緣運算與資料傳輸裝置
	+ 14TB可用空間
+ AWS Snowball Edge
	+ Snowball Edge Storage Optimized
		+ 適用於大規模資料遷移、==週期性傳輸工作流程==
		+ 80TB HDD
		+ 1TB SSD
	+ Snowball Edge Compute Optimized
		+ 為==機器學習、影片分析、本機運算堆疊等，提供強大運算資源==
+ AWS Snowmobile
	+ 用來將==大量資料==傳輸到AWS
	+ 100PB容量
